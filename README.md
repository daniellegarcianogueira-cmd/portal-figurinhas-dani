Portal Oficial de Figurinhas – Bonequinhas da DaniEste documento é o seu guia completo para configurar e implantar o PWA (Progressive Web App) de figurinhas da sua comunidade. Siga cada passo com atenção para garantir que tudo funcione perfeitamente.✅ Entrega do ProjetoConforme solicitado, aqui estão todos os arquivos gerados para o projeto. Você pode baixá-los no final desta entrega.•index.html: A estrutura principal da página.•styles.css: O arquivo de estilização com o tema roxo/pink.•app.js: Toda a lógica de funcionamento do portal (conexão com Supabase, upload, galeria, etc.).•config.js: Arquivo de configuração onde você irá inserir suas chaves do Supabase.•manifest.webmanifest: Arquivo que define o comportamento do PWA.•sw.js: O Service Worker que permite o funcionamento offline e cache.•stickers_table.sql: O script SQL para criar sua tabela e configurar as regras de segurança.•/icons/icon-192.png: Ícone do app no tamanho 192x192.•/icons/icon-512.png: Ícone do app no tamanho 512x512.🚀 Passo a Passo: Do Zero à ImplantaçãoSiga as seções abaixo para configurar o backend no Supabase e hospedar seu site gratuitamente no GitHub Pages.Passo 1: Configuração do Supabase (Backend)O Supabase será nosso banco de dados e armazenamento de imagens. O plano gratuito é mais que suficiente para começar.1.1. Criar Conta e Projeto no Supabase1.Acesse o site supabase.com e crie uma conta gratuita (pode usar sua conta do GitHub para facilitar).2.Após o login, você será direcionado ao seu Dashboard. Clique em New Project.3.Escolha a organização (geralmente seu nome de usuário) e dê um Nome ao seu projeto (ex: portal-figurinhas-dani).4.Gere uma Senha para o banco de dados (anote-a em um lugar seguro, embora não a usaremos diretamente no código) e escolha a Região mais próxima de você (ex: South America (São Paulo)).5.Clique em Create new project e aguarde alguns minutos enquanto o Supabase prepara tudo para você.1.2. Obter as Chaves da APICom o projeto criado, você precisa das chaves que conectarão seu site ao Supabase.1.No menu lateral esquerdo, clique no ícone de engrenagem (Project Settings).2.Clique na seção API.3.Você verá duas informações importantes na seção Project API keys:•Project URL: É a sua SUPABASE_URL.•Project API Keys (anon public): É a sua SUPABASE_ANON_KEY.4.Copie esses dois valores. Nós os usaremos no config.js mais tarde.1.3. Criar a Tabela stickersAgora, vamos criar a tabela que armazenará as informações de cada figurinha.1.No menu lateral esquerdo, clique no ícone de tabela (Table Editor).2.No painel que se abre, clique em New table. Mas vamos fazer de um jeito mais fácil e completo!3.Clique no ícone de terminal SQL (SQL Editor) no menu lateral.4.Clique em + New query.5.Copie todo o conteúdo do arquivo stickers_table.sql (que está nesta entrega) e cole no editor de SQL.6.Clique no botão verde RUN.Isso fará o seguinte, de uma só vez:•Criará a tabela stickers com todas as colunas corretas.•Ativará a Row Level Security (RLS), uma camada de segurança fundamental.•Criará as Policies (regras) que permitem que qualquer pessoa leia os dados (SELECT) e insira novos dados (INSERT), mas não permite que alterem (UPDATE) ou apaguem (DELETE) nada. Isso é crucial para a segurança!1.4. Criar o Bucket de Armazenamento (Storage)É aqui que as imagens das figurinhas serão guardadas.1.No menu lateral esquerdo, clique no ícone de imagem (Storage).2.Clique no botão Create a new bucket.3.Preencha os campos:•Bucket name: stickers (tem que ser exatamente este nome).•Marque a caixa This bucket is public.4.Clique em Create bucket.5.Agora, precisamos configurar as regras de acesso para o bucket. Volte ao SQL Editor.6.Cole apenas a parte de configuração do Storage do arquivo stickers_table.sql (a partir de -- CONFIGURAÇÃO DO STORAGE) e clique em RUN novamente. Isso garantirá que todos possam ver as imagens, mas o upload seja controlado.Pronto! Seu backend no Supabase está 100% configurado e seguro.Passo 2: Configuração do Projeto (Frontend)Agora, vamos conectar o site que eu criei com o seu backend Supabase.1.Abra o arquivo config.js em um editor de texto.2.Você verá as seguintes linhas:Copiarconst SUPABASE_URL = 'https://SEU_PROJECT_ID.supabase.co';
const SUPABASE_ANON_KEY = 'SUA_ANON_KEY_AQUI';
3.Substitua os valores 'https://SEU_PROJECT_ID.supabase.co' e 'SUA_ANON_KEY_AQUI' pelas chaves que você copiou no Passo 1.2.4.Opcional: No mesmo arquivo, você pode mudar o COMMUNITY_CODE para o código secreto que sua comunidade usará.É só isso! O resto dos arquivos já está pronto para funcionar.Passo 3: Deploy no GitHub PagesVamos colocar seu site no ar, de graça, usando o GitHub.1.Crie uma conta no github.com se ainda não tiver.2.Crie um novo repositório. Dê o nome que preferir (ex: portal-figurinhas). Marque-o como Público.3.Faça o upload de todos os arquivos do projeto para este repositório. Certifique-se de manter a estrutura de pastas (a pasta icons deve estar no mesmo nível que o index.html).4.Com os arquivos no repositório, vá para a aba Settings.5.No menu lateral de Settings, clique em Pages.6.Na seção Branch, selecione a branch main (ou master) e a pasta / (root). Clique em Save.7.Aguarde alguns minutos. O GitHub irá construir e publicar seu site. A página será atualizada com o link público do seu portal (algo como https://SEU_USUARIO.github.io/NOME_DO_REPOSITORIO/).Parabéns! Seu PWA está no ar e acessível para todos!Passo 4: Instalação e Teste4.1. Instalando o App no Celular1.Abra o link do seu site (que você pegou no GitHub Pages) no navegador do seu celular (Chrome no Android, Safari no iOS).2.No Android (Chrome): Um banner deve aparecer na parte inferior da tela sugerindo "Adicionar à tela inicial". Se não aparecer, clique nos três pontinhos do menu do Chrome e escolha Instalar aplicativo.3.No iOS (Safari): Clique no ícone de compartilhamento (um quadrado com uma seta para cima) e depois em Adicionar à Tela de Início.O ícone do app aparecerá na sua tela inicial, e ele abrirá como um aplicativo de verdade, sem a barra de endereço do navegador.4.2. Testando o Upload1.Abra o app.2.Clique em Adicionar Figurinha.3.Selecione uma imagem PNG ou WebP do seu celular.4.Preencha a categoria e o código da comunidade (o que você definiu em config.js).5.Clique em Enviar Figurinha.Se tudo deu certo, a modal fechará e sua figurinha aparecerá na galeria quase que instantaneamente!🚨 Solução de Problemas (Troubleshooting)| Problema | A figurinha não aparece após o upload.              | 1.  Verifique as chaves no config.js: Um único caractere errado impede a conexão. Copie e cole novamente.
2.  Verifique o Console do Navegador: Aperte F12 (no PC), vá na aba "Console" e veja se há erros em vermelho. Erros como 401 Unauthorized ou Bucket not found indicam problemas de configuração no Supabase.
3.  Policies (RLS): Certifique-se de que você executou o script SQL completo e que as policies de SELECT e INSERT foram criadas tanto para a tabela quanto para o bucket.
4.  Nome do Bucket: Confirme se o nome do bucket no Supabase é exatamente stickers.
| Erro ao copiar para o Instagram.                    | A função navigator.clipboard.write() é moderna e possui restrições de segurança.1.HTTPS é Obrigatório: O GitHub Pages já fornece HTTPS, então isso não deve ser um problema lá. Em testes locais (localhost), pode não funcionar.2.Suporte do Navegador/OS: Funciona melhor no Chrome para Android (versão 76+). Em alguns celulares ou versões do Instagram, a opção "Colar" pode não aparecer. Isso é uma limitação da plataforma, não do código.3.Ação do Usuário: A cópia deve ser iniciada por um clique do usuário. O código já faz isso.4.Alternativa: Se a cópia falhar, o app orienta o usuário a usar o botão Salvar e depois colar manualmente no Instagram a partir da galeria do celular. Essa é a alternativa mais robusta.
| O site não fica offline / não instala.              | 1.  Service Worker: Verifique no Console do navegador (aba "Application" -> "Service Workers") se o sw.js está ativo e rodando.5.HTTPS: O PWA só instala em conexões seguras (HTTPS), o que o GitHub Pages já oferece.6.Limpar Cache: Às vezes, o navegador guarda versões antigas. Tente limpar o cache do navegador para o site e recarregar.💖 ConclusãoSeu portal de figurinhas está pronto! Ele foi construído para ser rápido, seguro, gratuito e fácil de usar.
